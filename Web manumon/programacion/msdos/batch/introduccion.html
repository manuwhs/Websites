Batch

Como ya hemos dicho antes, MS-DOS posee un conjunto de comandos que constituye su lenguaje,
este conjunto es conocido como Batch y con él podemos manipular este sistema operativo.
Un comando es una instrucción o mandato que el usuario proporciona a un sistema informático, desde la línea de comandos.

Los archivos que contienen este lenguaje son archivos batch y poseen la extension .bat. Son archivos de procesamiento por lotes.
Se trata de archivos de texto sin formato, guardados con la extensión *.bat que contienen un conjunto de comandos DOS.
Cuando se ejecuta este archivo bat, los comandos contenidos son ejecutados en grupo, de forma secuencial,
permitiendo automatizar diversas tareas. Cualquier comando DOS puede ser utilizado en un archivo batch.

Hay que aclarar Batch que no es un Lenguaje de Programación.
Es un archivo de código que contiene comandos del, Shell de Windows, llamado MS-DOS.
Estos comandos se pueden ejecutar desde Inicio --> Ejecutar --> CMD o creando un archivo .bat con comandos.

Por tando el lenguaje batch sirve para ejecutar comandos MS-DOS del Shell de Windows.

Este lenguaje esta formado fundamentalmente por comandos, pero como en otros lenguajes, podremos definir variables
y escribir sentencias condicionales, pero solo en caso de que creemos un archivo .bat, de la otra manera no se podria.

Como el resto de los tutoriales, escribiremos este lenguaje en un documento de texto le cambiaremos la extension.
Con referencia a la sintaxis, las variables de definen con "set" y cuando queramos nombrarlas tenemos que poner su nombre entre %.

Para poner comentarios a un código solo basta usar el comando REM seguido del comentario, o con dos veces 2 puntos o con signos porcentaje: REM[comentario] o ::[comentario]

A proposito, cuando veamos por ejemplo la estructura de los diferentes comandos, los atributos que tenemos que poner nosotros
apareceran entre "[]", pero estos no se ponen, por ejemplo para poner un comentario seria asi:
:: esto es un comentario y estaria mal ponerlo asi:
:: [esto es un comentario]

Algunos de los comandos mas normales son:
echo [contenido]: Este comando muestra en la ventana shell de Window el [contenido]
pause: Este comando para la lectura del archivo .bat ya que por ejemplo, si usamos el comando anterior
y no ponemos pause, el lector del archivo seguira leyendo, y la ventana shell se cerrara.
exit: Este comando sirve para salir y cerrar el archivo sin que se pueda leer la continuacion

Cuando creemos un archivo batch, los comandos que escribamos se ejecutaran de la misma manera que lo harian en el Shell.
Me explico, si tu abres el Shell de Windows pondra "C:\Documents and Settings\Usuario>" que es la direccion de tu cuenta
de usuario y el simbolo ">" que significa que es la direccion de entrada para los comandos que escribamos a continuacion.
Ejemplo:
Si abrimos el shell y escribimos el comando "dir", nos mostrara todos los directorios que tenga la carpeta que este escrita,
la que viene de serie al abrir es "C:\Documents and Settings\Usuario".
Si en un archivo batch escribimos "dir" hara exactamente lo mismo, pero deberemos escribir tb pause para que no siga leyendo.

Otra cuestion que se explicara mas adelante detenidamente es el redireccionamiento ">" que hace que lo que este a la izquierda
sirva de informacion de entrada para lo que esta a la derecha, que en el caso anterior es un comando "dir", que muestra
los directorios de la direccion, pero si escribimos "dir>directorios.txt", a su vez, la informacion de los directorios servira de entrada
para el archivo directorios.txt lo que hara que es cree el archivo "directorios.txt" y que en su interior esten los directorios de la direccion.
