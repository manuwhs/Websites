Variables

# Una variable es un espacio en memoria donde se almacena un dato, un espacio donde podemos guardar cualquier tipo de información que necesitemos para realizar las acciones de nuestros programas.

# Hay cuatro clases de variables en archivos de batch:
1- Argumentos de línea de mando.
2- Variables de ambiente.
3- ERRORLEVELs.
4- FOR variables. .
Todos excepto ERRORLEVEL, que es de tipo "byte", son de tipo "String".

Argumentos de línea de mando

Son las variables que ponemos despues del nombre del archivo variables.

Variables de ambiente

# Son aquellas variables que residen en el ambiente corriente del área de memoria.
Es decir, son contenedores de informacion que definimos nosotros dentro del archivo .bat.
Declarar variables consiste en definir y de paso informar al sistema de que vas a utilizar una variable.
En bat, para definir una variable solo necesitamos anteponerle la particula "set".
Y para nombrarla despues otra vez tenemos que ponerla entre %.
Su estructura es esta: set nombre_variable = valor.
Ejemplo:
set numero = 5
echo %numero%
pause
 Aqui definimos una variable mediante set y luego la mostramos en pantalla mediante echo, para que se quede mostrado y no se cierre escribimos el comando pause. Esto se explicara con detalla en otros temas.

# Estas variables pueden tener otro parametro que puede ser o /p o /a. Veremos ahora lo que hacen ambos:
/p : Esta particula puesta antes del nombre de una variable, hace que se abra la ventana Shell mostrando el contenido de
la variable y un espacio para escribir, lo que escribamos sera igual a la variable.
Su estructura es: set /p nombre_variable /p = texto_inicial.
Ejemplo:
@echo off
set /P var=tu nombre :
echo Usted se llama %var%
pause
Aqui aparece un nuevo comando que aun no hemos mencionado, el "@echo off" este comando hace que no se muestre el eco.
En este codigo definimos una variable "var" en la cual al poner la particula /p, su valor sera igual al valor que intruduzcamos cuando se abra la ventana Sell, y el texto que precedera a esa introduccion de texto que haremos sera el valor que le hallamos asignado a la variable cuando la definimos.

/a : Esta particula sirve para hacer calculos, en batch por ejemplo si sumamos dos variables que son numeros, no dara la suma de los dos numeros sino "var1 + var2", este paramatro sirve para provocar que se ejecute la operacion.
Su estructura es: Su estructura es: set /a nombre_variable = operacion
Ejemplo:
@echo off
set numero=12
set numero2=18
set /a resultado=%numero%+%numero2%
echo %resultado%
pause

# Hay otra forma de variables de entorno, son variables que cuyo valor se indica cuando lo ejecutamos, es decir para acceder a ellas, ejecutamos el archivo y al hacerlo indicamos seguido del nombre del archivo, las variables que usaremos.
Esto presenta una gran ventaja ya que podremos definir estas variables con el valor que queramos indicandolo cuando lo ejecutemos.
Pero para poder usarlas tenemos que usarlas tenemos que ejecutar desde el Sell de Windows ya que es aqui donde nos permitira dar valor a las variables. Ejemplo:
C:\WINDOWS\> archivo.bat gato perro ave
 Aqui ejecutamos el archivo "archivo.bat" que esta en la carpeta windows y tambien definimos tres variables gato, perro y ave.
Para llamar estas variables en el codigo interior usaremos la notacion %numero.
Es decir para llamarlas escribiremos % seguido del numero que ocupe la variable.
Por ejemplo en el ejemplo anterior:
* gato = %1
* perro = %2
* ave = %3
La variable %0 es igual a la direccion del archivo .bat que estamos ejecutando.

# Hay unas variables de ambiente exportadas que están siempre disponibles, es decir variables ya creadas:
PATH: "Direccion"
El path de un archivo o unidad de disco es su direccion. MS-DOS nos proporciona la lista de archivos del ordenador para que nosotros podamos buscarla, encontrarla y ejecutarla.
