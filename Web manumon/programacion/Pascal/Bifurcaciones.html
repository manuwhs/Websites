Bifurcaciones

Bifurcaciones condicionales

Las estructuras condiciones permiten determinar si se da una determinada situacion y,
seguidamente ejecutar las instrucciones que se especifiquen.
De este tipo son las siguientes estructuras.

IF...THEN...ELSE

El IF se traduce como el "si" condicional, el THEN como "entonces" y el ELSE como "en caso contrario".
Esta expresión es utilizada para ejecutar una sentencia en el caso que una condición establecida sea verdadera,
de lo contrario se podra ejecutar una sentencia distinta.
Su estructura es esta:
IF condición THEN
BEGIN
instrucciones;
END

ELSE
BEGIN
instrucciones;
END;
Esta estructura nos dice que si "condicion" es verdadera, se realizaran las instrucciones que van despues de THEN,
si es falsa, se realizaran las que van despues de ELSE.
Como condicion podemos poner una comparacion de 2 valores, una variable booleana...

Si os fijais, el END del grupo de instrucciones que va despues de THEN, no lleva punto y coma ";",
esto es debido a que si lo ponemos daremos por finalizada la estructura y el ELSE nos dara error,
si no pusieramos la parte del ELSE, si tendriamos que poner el punto y coma ";".
Dentro de ese primer grupo de instrucciones, las instrucciones que lo forman si van separadas por ";".
Recordad que si solo queremos ejecutar una instruccion no hara falta ponerla entre BEGIN y END
ya que estas son para agrupar un grupo de instrucciones.
En tal caso, la instruccion que va despues del THEN tampoco llevaria punto y coma ";".

Program IF_THEN_ELSE;
VAR
Contador : Integer;
BEGIN
contador:=1;
IF contador=10 THEN
BEGIN
writeln('contador es igual a 10');
END
ELSE
BEGIN
contador:= contador + 1;
writeln('contador es menor a 10');
END;
END.

Tambien podemos combinar esta estructura condicional con un ELSE de esta estructura para decir que
si la condicion principal es falsa y pero otra condicion es verdadera ejecutar esa accion, su estructura seria esta:
IF condición THEN
BEGIN
instrucciones;
END

ELSE IF condicion2 THEN
BEGIN
instrucciones;
END
ELSE
BEGIN
instrucciones;
END;

CASE

Esta forma de control se utiliza cuando se va a evaluar una expresión, variable o constante
que puede contener varios datos diferentes (valores) y en cada dato deberá realizar una acción especial.
Esta estructura puede ser utilizada para variables, pero es mucho mas util para Arrays (matrices).
Su estructura es esta:
CASE selector OF
      valor1 : sentencia1;
      valor2 : sentencia2;
              ...
      valorN : sentenciaN;
      ELSE sentencia_else;
   END;
Esta estructura funciona de la siguiente manera:
Tenemos una expresion, el "selector" que puede ser una variable, contastante o expresion.
Y lo que hacemos es ir provando valores, si el "selector" tiene el "valor1" entoces se ejecutara la "sentencia1",
si el selector tiene el "valor2" se ejecutara la "sentencia2".
Al final de cada sentencia para cada valor, escribimos punto y coma ";".
Cuando hallamos puesto todos los valores que queramos probar, podemos poner una sentencia
que diga que si el "selector" no tiene ninguna de esos valores, entonces se ejecute,
esta sentencia es la que va despues del ELSE, la "sentencia_else".
Para terminar la estructura escribimos END; con " ; ", este END no tiene BEGIN.

PROGRAM 1Case;
VAR
Numero : Integer;
BEGIN
WriteLn('Introduzca un número entero del 1 al 5: ');
ReadLn(Numero);
CASE Numero OF
1 : WriteLn('El número fué 1');
2 : WriteLn('El número fué 2');
3 : WriteLn('El número fué 3');
4 : WriteLn('El número fué 4');
5 : WriteLn('El número fué 5');
END;
readln;
End.

Bifurcaciones incondicionales

Son aquellas que modifican la lectura del programa sin ninguna condicion. Existen dos tipos:

GOTO

La sentencia GOTO es una sentencia utilizada para alterar el flujo del programa, es decir,
para ir a ejecutar una sentencia en un lugar diferente del programa y no la linea siguiente.
Para utilizar este tipo de salto es necesario declarar etiquetas, que no son otra cosa que
el identificador que marcará el lugar a donde se dirigirá el flujo del programa al momento de usar el GOTO.
La declaración de etiquetas se hace antes que la de constantes y variables,
la palabra reservada para su declaración es LABEL.
El nombre de la etiqueta es un nombre de un identificador como cualquier otro, pero al utilizarse debe terminar con dos puntos ":".
Para dirigirnos a la etiqueta usaremos esta estructura: GOTO etiqueta;

PROGRAM Uso_del_GOTO;
LABEL
Etiqueta1;
BEGIN
WriteLn('Esta linea si se escribirá');
GOTO Etiqueta;
WriteLn('Esta linea no se escribirá');

Etiqueta1:
WriteLn('Se efectuó el brinco');
END.

HALT

La instrucción HALT provoca la terminación inmediata de un programa, sin importar las siguientes instrucciones.
Normalmente se utiliza en casos de error, se despliega un mensaje y se termina el programa. Ejemplo:

PROGRAM Uso_de_HALT;
BEGIN
WriteLn('Primera linea del programa');
WriteLn('Terminará la ejecución del programa');
HALT
END.
