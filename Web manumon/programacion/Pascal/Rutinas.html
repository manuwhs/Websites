Rutinas (subprogramas)

Rutinas

Una rutina es una serie de instrucciones con un único nombre (identificador),
que pueden ser activadas muchas veces utilizando el mismo.
Las rutinas cumplen la funcion de encapsular codigo y hacerlo mas manejable y accesible.
En lugar de reescribir el código completo de esa rutina cada vez que se necesite,
unicamente se hace una referencia a la rutina mediante las llamadas a procedimientos y funciones.
Existen tambien un monton de procedimientos y funciones en las bibliotecas de las unidades,
que podemos utilizar llamando a su keyword (identificador).
En Pascal, una rutina puede asumir dos formas: un procedimiento y una función.

En teoría:
* Un procedimiento es un conjunto de instrucciones que realizan una tarea concreta para resolver un determinado problema.
* Una función es un conjunto de instrucciones que realizan una tarea y devuelve un valor.
Por lo que esta dos rutinas de diferencian en que una función tiene un resultado, un valor de salida, y un procedimiento no.
Ambos tipos de rutinas pueden tener múltiples parámetros, de tipos de datos dados; y realizar tareas.

En la práctica:
La diferencia entre funciones y procedimientos es muy limitada:
* Se puede hacer una llamada a una función para realizar cierta tarea y luego saltarse el resultado
(que podría ser un código de error opcional o algo similar)
* O se puede hacer una llamada a un procedimiento que transmite un resultado dentro de sus parámetros.

Como veis la diferencia entre funcion y procedimiento es que las funciones devuelven un valor siempre,
pero este valor muchas veces es obviado ya que usamos las funciones para resolver problemas, igual que los procedimientos.

Procedimientos

Las funciones y procedimientos se declaran antes del cuerpo principal del programa y despues de haber declarado
el encabezado, las unidades, constantes, tipos y variables del programa.
La estructura de declaracion mas simplre de un procedimiento es la siguiente:

procedure [titulo];
begin 
              [instrucciones]
end;

Como veis empieza con la keyword procedure, que sirve para declarar un procedimiento, seguida del nombre del procedimiento,
este nombre se convierte en una keyword (identificador), que nos servira para llamar al procedimiento.
Despues escribimos las instrucciones del procedimiento entre las keywords "begin y end;".
Como veis, el "end" de los procedimientos no termina en punto "." sino en punto y coma ";".

Como si se tratara del cuerpo principal escribimos las intrucciones, esto quiere decir que dentro de un
procedimiento podemos llamar a otros procedimientos y declarar variables que seran "locales".
Los procedimientos se llaman como dijimos en el tema de las instrucciones simples, mediante su nombre,
es nombre tambien puede ir acompañado de parametros, cosa que daremos mas adelante en este tema.
Aqui teneis un ejemplo de procedimiento:

program procedimiento;
procedure titular;
begin
WriteLn ('Programa de Turbo Pascal');
readln;
end;
begin
titular; (*llamada al procedimiento*)
end.

Parametros

Para poder pasar información entre el programa principal y procedimientos, o entre procedimientos usamos los parámetros.
Los parametros son canales de comunicación de datos, son variables que definimos al llamar al procedimiento,
y que desepeñaran una funcion dentro de este.
Los parámetros son opcionales y si no se necesitan no se deben usar.

Para utilizarlos es necesario declararlos son la siguiente sintaxis:
PROCEDURE nombre (lista de parámetros);

La lista de parámetros esta compuesta de los nombres de los mismos y del tipo de datos que representan,
los del mismo tipo se separan con comas "," y cada tipo diferente se separa con punto y coma ";". Ejemplo:
Procedure Ejemplo(a, b : Integer; c, d : String);

Para llamar a un procedimiento que utiliza parámetros se pueden utilizar como tales otras variables o constantes,
siempre y cuando sean del mismo tipo que los declarados.
Para darle un valor a los parametros, al llamar al procedimiento, debemos escribir el valor de dichos parametros
entre parentesis y el primer valor que pongamos equivaldra al primer parametro que hallamos definido, el 2º al 2º.....
Vamos a llamar al procedimiento que hemos definido antes y le vamos a dar valores:
Ejemplo(1, 2, 'Hola', 'Adios');

Una vez sabido esto crearemos un ejemplo con parametros:

PROGRAM Parametros;
VAR
Saludo : String;

PROCEDURE Imprime_5 (a : String);
VAR
Contador : Integer;
BEGIN
FOR contador := 1 to 5 DO 
{Imprime 5 veces la cadena almacenada }
WriteLn(a); 
{en la variable "a", que es la información}
END; 
{que llega como parámetro }

BEGIN
Saludo := 'Bienvenidos al programa';
Imprime_5 (Saludo); 
{llama al procedimiento Imprime_5, usando como}
{parámetro la variable Saludo }
Imprime_5 ('Fin'); 
{Utiliza la cadena "fin" como parámetro }
END.

Funciones

Las funciones son, al igual que los procedimientos, un conjunto de sentencias que se ejecutan constantemente, la diferencia entre éstas y los procedimientos es que las funciones regresan un valor. La declaración de las funciones se realiza de la siguiente forma: 

FUNCTION [titulo](parámetros) : tipo_de_datos; 
begin 
              [instrucciones]
end;

Donde [titulo] sera el identificador, "parametros", los parametros, al igual que en los procedimientos
y "tipo_de_datos" el valor que devolvera la varriable.
PROGRAM Funciones;
VAR
X, Y, Z : Real;

FUNCTION Promedio (a, b : Real) : Real;
{Promedio de dos números reales}
BEGIN
Promedio := (a + b) / 2;
END;

BEGIN
X := 5.89;
Y := 9.23;
Z := Promedio (X, Y);
{Iguala Z al valor devuelto por la función Promedio}
WriteLn('El promedio de ',X,' y ',Y,' es: ',Z);
END.
