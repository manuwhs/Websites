Unidades

Turbo Pascal permite el acceso a un número considerable de constantes, tipos de datos, variables,
procedimientos y funciones predefinidos. Como son muchos y en un programa es muy raro que se utilicen todos,
están repartidos en grupos relacionados entre sí llamados Unit (Unidades).

Una Unidad es una librería de declaraciones que se pueden introducir en un programa
y que permite dividir el código y compilarlo por separado, es decir:
Una Unidad es un conjunto de constantes, tipos de datos, variables, procedimientos y funciones predefinidos.
Cada unidad es “casi” un programa en Turbo Pascal, normalmente un conjunto de datos, tipos de datos
y subprogramas que pueden usarse desde otro programa.
Pascal divide sus funciones, procedimientos... predefinidos en librerias para que los programas se ejecuten mas rapido,
ya que si no vamos a usar un tipo de declaraciones, es mejor que no se compilen para que no ocupen tiempo de lectura.

Declaracion

Para declarar una unidad, o lo que es lo mismo, dejarla lista para ser utilizada, se utiliza la keyword USES
que debe ir inmediatamente despues de la sentencia PROGRAM, seguida de los nombres de las unidades que se utilizarán.
De esta manera ----> USES nombre_unidades
Las unidades estándar de turbo pascal son:
Crt, Dos, Graph, Overlay, Printer, System.
Estas unidades las daremos en los siguientes apartados de este tema.

Creacion

Podemos crear nuestras propias Unidades, compilarlas y utilizarlas en otro programa.
Las unidades estan compuestas por cuatro partes, para crearlas, deberemos definirlas;

Declaracion

Es obligatoria, provee el nombre de la unidad que se creará. Su suntaxis es:
UNIT nombre_de_la_unidad;

Ejemplo:
UNIT prueba;

Interfaz

Esta sección se declara con la palabra INTERFACE, en esta sección se encuentran las declaraciones de todos
los elementos que pueden ser utilizados por el programa que llama a la unidad, por ejemplo otras unidades, variables,
constantes, procedimientos y funciones.
En esta sección no se incluyen los códigos de los procedimientos ni funciones, unicamente su declaración. Esta es la parte declarativa de la Unidad, Ejemplo:

INTERFACE
USES Crt, Dos;
VAR
Numero : Integer;
CONST
Columnas = 10;
PROCEDURE Colocar(Cad1 : String; X, Y : Integer);

En este ejemplo, la Unidad que tenga este interfaz poseera la libreria de las unidades Crt y Dos,
ademas de la variable predefinida Numero, la contante predefinida Columnas y el procedimiento
predefinido Colocar.

Implementacion

La sección de implementación es exclusiva para la unidad que se está creando,
contiene declaraciones de etiquetas, variables, constantes, así como el contenido de los procedimientos y funciones
declarados en la sección de interfaz. La palabra usada para declarar esta parte es IMPLEMENTATION.
IMPLEMENTATION
USES Crt;
PROCEDURE Colocar(Cad1 : String; X, Y : Integer);
BEGIN
GotoXY(X,Y);
Write(Cad1);
END;

Como se puede observar tambien se ha declarado aqui la unidad Crt, mediante "USES Crt".
Esto se ha hecho asi porque necesitabamos usar una funcion que posee esa unidad dentro del procedimiento "colocar".
Al declarar una unidad aqui solo se declararan las funciones que usemos en esta parte,
y no todas como sucede si lo declaramos en la parte de INTERFACE.

Inicialización

Esta sección se utiliza para inicializar los valores de las variables, no es indispensable y por lo mismo no siempre se incluye.
Su sintaxis es parecida a la del cuerpo principal de un programa, se inicia con BEGIN y termina con un END.


Asi pues, para crear una Unidad debemos definir las 4 partes anteriores. Ejemplo:
UNIT Prueba;
INTERFACE
PROCEDURE Centrar(Cad1 : String; renglon : Integer);
IMPLEMENTATION
USES Crt;
VAR C : Integer;

PROCEDURE Centrar(Cad1 : String; renglon : Integer);
BEGIN
C := (80 - Length(Cad1)) DIV 2
GotoXY (C,renglon);
Write(Cad1);
END;

BEGIN
END.


Compilacion y uso de la Unidad

Cuando tengamos lista nuestra unidad, la compilamos en el Disk, no en Memory, y se creara un archivo .TPU.
Una vez compilada una unidad se puede incorporar al programa agregando la sentencia USES
de la que se había hablado al principio, ya declaradas las unidades que se utilizarán se puede hacer uso de todos
los procedimientos y funciones incluidas como si estuvieran escritas en el mismo programa.
