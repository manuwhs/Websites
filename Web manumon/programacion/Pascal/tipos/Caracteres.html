Tipos

El tipo de un dato es el conjunto de valores que puede tomar durante el programa.
Si se le intenta dar un valor fuera del conjunto se producirá un error.
El tipo de una variable es entonces el conjunto de valores que puede tomar durante el programa.
La asignación de tipos a los datos tiene dos objetivos principales:
* Detectar errores en las operaciones
* Determinar cómo ejecutar estas operaciones
Pascal es un lenguaje fuertemente tipeado, esto quiere decir que todos los datos deben de tener un tipo declarado
explícitamente, y además que existen ciertas restricciones en las expresiones en cuanto a los tipos de datos que en ellas intervienen.

Tipos de datos

1.1 Dinamicos

Dentro de esta categoría entra sólamente el tipo puntero.
Este tipo te permite tener un mayor control sobre la gestión de memoria en tus programas.
Con ellos puedes manejar el tamaño de tus variables en tiempo de ejecución, o sea, cuando el programa se está ejecutando.
Es decir, se puede cambiar el tamaño que ocupa un tipo de dato en memoria.

1.2 Estaticos

Casi todos los tipos de datos son estáticos, la excepción son los punteros y no se tratarán debido a su complejidad.
Que un tipo de datos sea estático quiere decir que el tamaño que ocupa en memoria
no puede variar durante la ejecución del programa.
Es decir, una vez declarada una variable de un tipo determinado,
a ésta se le asigna un trozo de memoria fijo, y este trozo no se podrá aumentar ni disminuír.

1.2.1 Ordinales

Un tipo es ordinal porque el conjunto de valores que representa se puede contar, es decir,
podemos establecer una relación uno a uno entre sus elementos y el conjunto de los números naturales.
Los tipos ordinales están basados en el concepto de orden o secuencia.
Usted no sólo puede comparar dos valores para ver cuál es más alto, sino que también puede pedir el
valor que sigue o precede a un valor dado, o calcular el valor más bajo o alto de los posibles

Por ejemplo el tipo 'numeros enteros', es ordinal porque se puede saber que numero va despues de otro,
el siguiente numero al 2 sabemos que es el 3, pero en el tipo 'real' no se sabe que numero va despues,
ya que valdria cualqiera: 2,13 o 2,135 o 2,1354.

1.2.1.1 El tipo integer (entero)

Al declarar una variable de tipo entero, estás creando una variable numérica que puede tomar valores positivos o negativos, y sin parte decimal. Este tipo de variables, puedes utilizarlas en asignaciones, comparaciones, expresiones aritméticas, etc. Ejemplo:

var
telefono, edad:integer;
begin
telefono := 654235689485;
edad := 22;
end.

1.2.1.2 El tipo boolean (lógico)

El tipo de datos lógico es el que te permite usar variables que disponen sólo de dos posibles valores: cierto o falso.
Debido a esto, su utilidad salta a la vista, y no es otra que variables de chequeo.
Nos sirven para mantener el estado de un objeto mediante dos valores, 0 y 1, que pueden indicar:
* si/no
* cierto/falso
* funciona/no funciona
* on/off
* etc.
Ejemplo:

var
conectado,encendido:boolean;
begin
conectado := 0 (*verdadero*)
encendido := 1 (*falso*)
if conectado and (not encendido) then
writeln('No esta en marcha')
else
writeln('Esta en marcha')
end.

La estructura condicional que hay en el ejemplo la daremos mas adelante, pero es facil de interpretar.

1.2.1.3 Los tipos char y string (caráctery cadena)

* Con el tipo carácter (char) puedes tener objetos que representen una letra, un número, un signo ...
Es decir, puedes usar variables o constantes que representen un valor alfanumérico.
Pero ojo, cada variable sólo podrá almacenar un carácter.
* Con las cadenas de caracteres (strings) puedes contener en una sóla variable más de un carácter.
Los valores de estos dos tipos deben ir entrecomillados, entre comillas.
Ejemplo:

var
hotmail:string;
letra:char;
begin
hotmail := 'manumon_20@hotmail.com';
letra := 'H';
end.

1.2.2 Tipos no-ordinales, TIpo real (real)

Simplificando, podríamos reducir los tipos simples no-ordinales al tipo real.
Este tipo nos sirve para declarar variables que pueden tomar valores dentro del conjunto de los números reales.
A diferencia de los tipos ordinales, los no-ordinales no se pueden contar.
No se puede establecer una relación uno a uno entre ellos y los número naturales.
Dicho de otra forma, para que un conjunto se considere ordinal se tiene que poder calcular la posición,
el anterior elemento y el siguiente de un elemento cualquiera del conjunto.¿Cuál es el sucesor de 5.12? Será 5.13, o 5.120, o 5.121

var
decimal,nazar:real; begin
decimal := 25.12; azar :=15,255; end.

Al mostrar este tipo de datos en pantalla, hay que indicar la cantidad de numeros enteros y decimales
que se mostraran. La estructura para hacerlo es: variable:enteros:decimales.
En el caso de este tipo de datos, el indicar la cantidad de numeros enteros carece de sentido ya que
se vera limitado por los numeros decimales, por lo que su valor sera 0 pero debemos indicar el
numero de decimales que se mostraran. Ejemplo:

program reales;
var
real1 : real;
begin
real1:=25.55;
writeln(real1:0:0);
writeln(real1:0:1);
writeln(real1:0:2);
writeln(real1:0:3);
writeln(real1:0);
readln;
end.
