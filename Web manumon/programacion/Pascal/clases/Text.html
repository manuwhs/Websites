Archivos de acceso secuencial

Los archivos de acceso secuencial son llamados también archivos de texto,
son aquellos archivos que están formados por cadenas de caracteres separadas
unas de otras por los códigos retorno de carro/avance de línea,
que corresponden a los caracteres 13/10 del código ASCII.
Es posible visualizar estos archivos con la orden TYPE del sistema operativo DOS y se pueden editar con cualquier editor de textos.
Son los archivos del block de notas, los archivos *.txt.
Este tipo de archivos solo permiten operaciones de lectura y escritura secuenciales,
la unica forma de modificarlos es agregando líneas al final del archivo.

Los pasos para leer o escribir en un archivo de este tipo son:
Abrir el archivo ya sea para lectura o escritura.
Leer o escribir datos en el archivo.
Cerrar el archivo

La Clase Text

La Clase Text sirve para declarar un archivo .txt, osea un archivo secuencial.
Este tipo de datos sirven para que se les asigne la direccion de un archivo secuencial (*.txt).
Es decir el contenido de estas variables sera el archivo secuencial que le asignemos.
Ejemplo de declaracion de una variable tipo Text:
Var
archivotexto : Text;

Una vez declarada, seguimos los pasos para leer o escribir un archivo:

Asignar el Archivo a la Variable

Una vez declarada una variables de tipo Text, hay que asignarle un archivo secuencial.
Esto se hace mediante el siguiente metodo:
* Assign(variable_text, direccion_archivo):
Mediante este metodo asignamos a la variable "variable_text" el archivo secuencial cuya direccion sea "direccion_archivo".
Siendo "variable_text" una variable tipo Text y direccion_archivo la direccion de un archivo secuencial. Ejemplo:
Assign(archivotexto, 'C:\PRUEBA.TXT'):

Abrir el Archivo

Una vez asignado el archivo a la variable hay que habrirlo, hay tres formas de abrirlo:
* Append(variable_text):
Este procedimiento habre el archivo para que pueda ser escrito.
* Rewrite(variable_text):
Este procedimiento abre el archivo para que pueda ser escrito y ademas elimina todo su contenido.
Ademas, si el archivo que pongamos en la direccion no existe, lo crea. * Reset(variable_text):
Este procedimiento abre el archivo para que pueda ser leido.

Escribir o Leer el Archivo

Una vez abierto, podemos escribir o leer, depende del procedimiento que hayamos usado para abrirlo:
# Para escribir en el archivo usamos los siguientes procedimientos:
* write(variable_text, Contenido):
Mediante este procedimiento, escribimos en al archivo de la variable "variable_text" el contenido que escribamos en "Contenido". Al acabar de escribir en el archivo, el puntero se coloca al final de la linea que hemos escrito.
* writeLn(variable_text, Contenido):
Mediante este procedimiento, escribimos en al archivo de la variable "variable_text" el contenido que
escribamos en "Contenido". Al acabar de escribir en el archivo, el puntero se coloca al principio de la
linea que hemos escrito.

# Para leer los archivos usamos los siguientes procedimientos:
* Read(variable_text, Linea):
Mediante este procedimiento, leemos una linea del contenido del archivo de la variable "variable_text"
y lo guardamos en la variable "Linea". Al leer la linea, deja el puntero en la misma linea que leyo los datos.
* Read(variable_text, Linea):
Mediante este procedimiento, leemos una linea del contenido del archivo de la variable "variable_text"
y lo guardamos en la variable "Linea". Al leer la linea, deja el puntero en la siguiente linea.

Si ejecutamos el procedimiento Read una vez, la variable linea, sera igual al primer renglon del archivo,
si lo ejecutamos dos veces, la variable linea tambien sera igual al primer renglon, porque el puntero se deja
en la misma linea; mientras que con ReadLn, el puntero se deja en el siguiente renglon por lo que si lo usamos
dos veces, la segunda vez, Linea sera igual al 2º renglon, si lo usamos 3, Linea sera igual al 3º renglon...

Para leer todas las lineas que tiene un archivo sin saber cuantas tiene, usaremos la sentencia WHILE combinado con
la funcion EOF(Archivo), esta funcion, comprueva si el puntero esta en la ultima linea del Archivo, si lo esta devuelve TRUE (1),
pero sino devuelve False (0), por eso, para leer y mostrar el contenido de un archivo secuencial usaremos esta estructura:
WHILE NOT EOF(Archivo) DO
BEGIN
ReadLn(Archivo, Datos);
WriteLn(Datos);
END;

Cerrar el Archivo

Para asegurar que toda la información se grabe en el disco correctamente es necesario cerrar el archivo
antes de que se termine la ejecución del programa, de lo contrario se corre el riego de que la última información
que se haya accesado no se grabe en su totalidad.
La instrucción para cerrar un archivo es Close, y su sintaxis es:
Close (Archivo);

Ejemplo

Para este ejemplo tendreis que crear un archivo llamado Texto.txt en el disco duro C:\

program secuenciales;
VAR
Archivo : Text;
Lineas : String;
Numero : Integer;

BEGIN
Assign(Archivo, 'C:\Texto.txt');
Rewrite(Archivo);
WriteLn(Archivo,' Este tutorial es una pasada ');
For Numero := 1 To 5 Do
Begin
writeLn(Archivo,Numero * 100);
end;
Close(archivo);

Reset(Archivo);
While not EOF(Archivo) DO
Begin
ReadLn(Archivo, Lineas);
writeLn (Lineas);
End;
Close(archivo);
ReadLn;
End.
