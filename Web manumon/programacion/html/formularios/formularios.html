Formularios


Un formulario es un elemento que permite recoger datos introducidos por el usuario. Los formularios se utilizan para conocer las opiniones, dudas, y otra serie de datos sobre los usuarios, para introducir pedidos a través de la red, tienen multitud de aplicaciones.
Un formulario está formado, entre otras cosas, por etiquetas, campos de texto, menús desplegables, y botones.

Los formularios se insertan a través de las etiquetas <form> y </form>. Entre dichas etiquetas habrá que insertar los diferentes campos de texto y botones para crear el formulario. Con HTML lo unico que podemos hacer es enviarlo por correo, asique tendremos que mandarlo procesar por otro lenguaje mas sofisticado. La etiqueta <form> tiene los siguientes atributos:

AtributoDescripcion Opciones
actionIndica el tipo de accion a llevar a cabo con el formulario.Enviarlo por correo o procesar con algun archivo.
enctypeIndica el modo en que queramos que se lea el archivo.Puede ser leido como texto plano, entoces su valor sería "text/plain" o puede ser ejecutado, en ese caso omitiremos esta etiqueta.
methodIndica la forma en la cual el formulario es enviado.Puede tomar como valor "get" o "post", el primero sirve para que no se puedan producir cambios como guardar el formulario y el segundo hace que si se puedan producir.

Vamos a explicar un par de cosas sobre la etiqueta action:
Cuando queremos que el formulario sea mandado por correo action sera igual a ---> action="mailto:formularios@fomulario.com"
Cuando queremos que el formulario sea procesado por algun archivo action sera igual a ---> action="direccion del archivo"
Ejemplos de formulario:
<form action="mailto:formularios@formulario.com" method="post" enctype="text/plain" > ... </form>
<form action="direccion.php" method="get"> ... </form>

Campos de texto

Una de las cosas más importantes de un formulario, son los campos de texto, en ellos es donde escribimos la informacion.
Para insertar un área de texto largo es necesario incluir las etiquetas <textarea> y </textarea> y para que sea leido por el formulario tiene que estar entre las etiquetas <form> y </form> del formulario. Esta etiqueta tiene varios atributos:

AtributoDescripcion
name Indica el nombre del campo de texto.
rows Indica el número de líneas que podrán ser visualizadas .
cols indica el número de caracteres por línea que podrán ser visualizado

Ejemplo de area de texto:
<form action="mailto:formularios@formulario.com" method="post" enctype="text/plain" > <textarea name="ejemploarea" cols="90" rows="5">Escribe el texto que quieras</textarea> </form>



Existe tambien otra etiqueta de introduccion de texto corto, la etiqueta <input>. Esta etiqueta tiene varios atributos:

AtributoDescripcion
name Indica el nombre del campo de texto.
type Indica el tipo de texto que visualizaremos al escribir, si su valor es "text", se ve lo que escribimos, si es "password" se ven puntos.
size indica el tamaño de la caja de texto en funcion de los caracteres.
maxlength Es el tamaño maximo de texto que puede ser tomado por el formulario
value Da un valor predefinido al campo de texto.


Ejemplo de area de texto:
<form action="mailto:formularios@formulario.com" method="post" enctype="text/plain">
<input type="password" name="contrasena" size=15 maxlength=10 value="contrasena"> </form>

Botones

Hemos visto la etiqueta <input>como campo de texto sin embargo esta etiqueta tambien sirve para crear botones.
Esto se logra gracias al atributo type="" que hemos visto antes:

Si el valor es submit, al pulsar sobre el botón se enviará el formulario.
Si el valor es reset, al pulsar sobre el botón se restablecerá el formulario, borrándose todos los campos del formulario que hayan sido modificados y adquiriendo su valor inicial.
Si el valor es button, al pulsar sobre el botón no se realizará ninguna acción.

Ejemplo de botones:
<form action="mailto:formularios@formulario.com" method="post" enctype="text/plain">
<input type="submit" name="contrasena" size=10 value="Enviar"> </form>



Tambien se pueden crear más tipos de botones modificando otra vez el atributo type:
Cuando es igual a checkbox se forma una caja de seleccion.
Cuando es igual a radio se forman se forma una caja de seleccion circular, para que solo una pueda ser selecciona tienen que tener el mismo name.

<input name="casilla" type="checkbox" value="acepto" checked>
<input name="prefiere" type="radio" value="estudiar" checked>
<input name="prefiere" type="radio" value="aprender">
<input name="prefiere" type="radio" value="mejorar">
<input name="prefiere" type="radio" value="algo">

Listas

Los campos de selección se utilizan para insertar menús y listas desplegables. Para insertar uno de estos menús o listas es necesario insertar las etiquetas <select> y </select>.
Estas etiquetas tambien tienen atributos como:
name: indica el nombre de la lista.
size: indica el numero de elementos de la lista que se muestran a la vez.
multiple: si aparece, se pueden seleccionar varios objetos de la lista alludado por la tecla Ctrl
disabled: si aparece indica que la lista estará desactivada.

Entre estas dos etiquetas, se va insertando <option> y </option> para indicar cada una de las opciones.
Mediante el atributo value="" indicaremos el valor a enviar en caso de que se elija esa opcion. Ejemplo de lista:
<form action="mailto:formularios@formulario.com" method="post" enctype="text/plain">
<select name="estaciones" size=1>
<option value="uno">Primavera</option>
<option value="dos">Verano</option>
<option value="tres">Otoño</option>
<option value="cuatro">Incierno</option>
</select>
</form>

Subir archivos

Si la etiqueta input tiene como tipo "file" ---><input type=file>
Entonces servira para enviar un archivo que buscaremos en nuestro PC a la web.
Ejemplo:
<form name="manda" action="subirmusica.php" enctype="multipart/form-data" method="post">
<input type=file name="archivo" id="archivo">
<input type=submit VALUE="Sube musica">
</form>

Deberemos tambien añadir un campo oculto para indicar el maxico tamaño del archivo en bytes:
<input type="hidden" name="MAX_FILE_SIZE" value="1000000">

Para poder manipular el archivo que nos envien, debemos saber PHP.
